# Engram Evaluation Harness — Gate E Baseline (2026-07-13)

## Overview

Two evaluation golden sets and a harness script under `evals/`:

- **Classification** (`evals/golden/classification_v1.json`): 20 samples covering
  all 8 builtin memory kinds (fact, preference, doctrine, decision, invariant,
  observation, procedure, summary). Each sample is a realistic memory write with
  an expected kind label.

- **Recall relevance** (`evals/golden/recall_v1.json`): 8 queries with relevant
  content fragments that should appear in top-K results. Measures precision@K,
  MRR, and recall@K.

## Baseline results (updated 2026-07-13)

### Classification — rule-only (no LLM, no built-in rules)

```
Accuracy: 20.0% (4/20)
```

### Classification — rule-only (with built-in rules)

```
Accuracy: 70.0% (14/20)
```

Built-in keyword rules for invariant, doctrine, procedure, summary, decision,
and observation. Invariant 0%→100%, observation 0%→100%, procedure 0%→100%,
summary 0%→100%.

### Classification — live (built-in rules + DeepSeek-V4-Flash via DeepInfra)

```
Accuracy: 85.0% (17/20)
```

Per-kind: fact 75%, preference 100%, decision 100%, observation 100%,
procedure 100%, summary 100%, doctrine 33%, invariant 50%.

3 remaining misses are doctrine↔invariant↔fact boundary cases.

### Model comparison

| Model | Provider | Accuracy | Elapsed | s/call |
|-------|----------|----------|---------|--------|
| DeepSeek-V4-Flash + rules | DeepInfra | **85%** | 126s | 6.3 |
| DeepSeek-V4-Flash (no rules) | DeepInfra | 80% | 126s | 6.3 |
| GPT-OSS-120B | OpenRouter | 65% | 179s | 9.0 |
| GPT-OSS-120B | DeepInfra | 60% | 209s | 10.4 |
| GPT-OSS-20B | DeepInfra | 60% | 133s | 6.7 |

### Recall — live on engram01 dogfood

```
Mean Precision@K: 0.025
Mean MRR:         0.062
```

Near-zero because the dogfood corpus contains only 2 items (the roadmap and a
probe). The recall eval requires a seeded corpus to produce meaningful numbers.
Seeding was blocked by a pre-existing DetachedInstanceError bug in the
classification vocab cache (see Known Issues below).

## Usage

```bash
# Rule-only classification (no instance needed)
python evals/run_evals.py --rules-only

# Full eval against a live instance
python evals/run_evals.py --base-url http://engram01:8000 --api-key eng_...

# JSON output for CI integration
python evals/run_evals.py --rules-only --json
```
